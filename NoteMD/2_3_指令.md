# 指令

## 渲染指令

### 纯文本渲染：`v-text`

```html
<tag v-text = "data"></tag>
```

不可被解析，即按照原样渲染到客户端；
[cmd_v_text.vue](../Code/vueint/src/components/2_3_commands/cmd_v_text.vue)

### 超文本渲染：`v-html`

```html
<tag v-html = "data"></tag>
```

可以被解析，即按照 HTML 格式渲染到客户端；
[cmd_v_html.vue](../Code/vueint/src/components/2_3_commands/cmd_v_html.vue)

## 属性绑定：`v-bind`

简写为 `:` 。

```html
<tag v-bind:attr = "data"></tag>
<!-- 或简写形式 -->
<tag :attr = "data"></tag>
```

动态绑定一个数据 `data` 到属性 `attr`。该属性属于标签 `tag` 。
也可以将 属性 和 字符串拼接表达式 绑定。如下：

```html
<tag :id = "'list' + index"></div>
```

这里把数据（一个 字符串拼接表达式）绑定到 `id` 上。

e.g.:
```html
<!-- 静态方式（每个用户都一样） -->
<img src="/default-avatar.jpg">

<!-- 动态方式（每个用户不同） -->
<img :src="user.avatarUrl">
```

此语段便可以解释为：
绑定 `user.avatarUrl` 到 `src` 属性，其中 `src` 属性属于 `img` 标签

[cmd_v_bind.vue](../Code/vueint/src/components/2_3_commands/cmd_v_bind.vue)

## 事件绑定：`v-on`

简写为 `@` 。
好熟悉啊，似乎不用额外记。
类比 Godot 的 `signal` 系统。
不对甚至不用类比到这么远的地方，JS 本身有这功能！

```html
<tag v-on:event = "func"></tag>
<!-- 或简写形式 -->
<tag @event = "func"></tag>
```

[cmd_v_on.vue](../Code/vueint/src/components/2_3_commands/cmd_v_on.vue)

## 双向数据绑定：`v-model`

为 `input`、`textarea` 和 `select` 元素创建双向绑定，即根据元素类型自动选取对应的属性 - 事件组合。

```html
<tag v-model = "data"></tag>
```
具体组合如下：
- `textarea` 和 `type="text"` 的 `input` 元素：
    绑定 `value` 属性和 `input` 事件
- `type="checkbox"` 的 `input` 元素：
    绑定 `checked` 属性和 `change` 事件
- `type="radio"` 的 `input` 元素：
    绑定 `checked` 属性和 `change` 事件
- `select` 元素：
    绑定 `value` 属性和 `change` 事件
