## 计算属性

[cp_computed.vue](../Code/vueint/src/components/2_6_calculate_property/cp_computed.vue)

其实就是计算界的函数封装。
这里的“计算”不是狭义的数字计算，而是包括了字符串、布尔等的，所有关于数据的变动的一类计算。

计算属性可以把一些复杂的计算流程封装进一个变量中。

与普通 method 不同的是，后者会在每次组件重新渲染时执行一次，但前者只会在需要的时候进行运算，性能更优。
但计算属性仅适用于从某个数据派生出变化的流程，如 `大于 60 分的元素变蓝色` ，或者 `60 + 80 * 50 需要频繁地算`。
如果流程涉及到“执行动作”，还是需要使用普通的 method。
至于如何区分二者：

- 你要自动地算吗？如果需要，使用 `computed`；
- 你要伺机行事吗？如果需要，使用 `watch`；
- 你要手动工作吗？如果需要，使用 `method`。

差不多就是这个区别。

需要先导入 vue 包， `{ computed }`

工作流程分为两步：

### 定义计算属性

使用 `computed()` 定义计算属性

```javascript
const name = computed(() => {
    return data
})
```

### 输出计算属性

在 HTML 部分通过 `{{ name }}` 展示计算属性。
